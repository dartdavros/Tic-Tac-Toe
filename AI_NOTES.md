## Stage 2: Глобальные стили и подключение в точку входа

### Что было сделано

- Создан `src/styles/global.css` со всеми 16 CSS-переменными в `:root`
- Применён CSS-сброс: `box-sizing: border-box`, `margin: 0`, `padding: 0`
- Тег `body` стилизован через переменные
- Добавлен медиазапрос `@media (max-width: 480px)`
- Обновлён `src/main.tsx`: добавлен `import './styles/global.css'`

### Почему такой подход

**Переопределение `--cell-size-desktop` в медиазапросе:** компонент `Cell`
использует единственную переменную. В медиазапросе она переопределяется
значением `80px`, что позволяет компонентам не менять ссылки.

**`--color-win-highlight: #f1c40f`:** жёлтый цвет семантически отличается
от акцентного зелёного `--color-accent`.

**`--cell-size-mobile` как справочное значение:** CSS не поддерживает
`var()` внутри `@media` условий, поэтому в медиазапросе используется
литеральное значение `80px`.

## Stage 3: Исправление ошибок сборки и тестов

### Что было исправлено

#### Ошибка 1: `vite.config.ts` — поле `test` не распознаётся

**Причина:** `defineConfig` из пакета `vite` не содержит типа для поля
`test` — оно принадлежит Vitest. TypeScript выдавал ошибку компиляции
`TS2769: No overload matches this call`.

**Решение:** Заменить импорт `defineConfig` с `'vite'` на `'vitest/config'`.
Пакет `vitest/config` реэкспортирует `defineConfig` из Vite, расширяя тип
конфигурации полем `test`. Это официально рекомендованный способ.

#### Ошибка 2: ESLint — переменные `_state` и `_dispatch` в `App.tsx`

**Причина:** Правило `@typescript-eslint/no-unused-vars` было настроено
с `argsIgnorePattern: '^_'`, который применяется только к **аргументам
функций**. Переменные из деструктуризации (`const [_state, _dispatch]`)
попадают под `varsIgnorePattern`, который не был задан.

**Решение:** Добавить `varsIgnorePattern: '^_'` в конфигурацию правила.
Это позволяет использовать префикс `_` для временных заглушек переменных.

#### Ошибка 3: Два падающих теста в `aiPlayer.test.ts`

**Причина:** Алгоритм minimax без учёта глубины рекурсии при одинаковой
оценке нескольких ходов выбирает первый найденный (итерация слева направо).

- Тест «выбирает центр на пустой доске»: на пустой доске при идеальной
  игре обеих сторон все ходы дают ничью (оценка `0`), поэтому возвращался
  индекс `0` (первый в списке), а не `4` (центр).

- Тест «ИИ за O выбирает победный ход»: ход `O` в клетку `2` (блокировка
  `X`) и ход в клетку `5` (немедленная победа `O`) при одинаковой глубине
  могли давать одинаковую итоговую оценку, и выбирался первый — `2`.

**Решение:** Добавить параметр `depth` в функции `evaluate` и `minimax`.
Оценка с учётом глубины:
- Победа ИИ: `10 - depth` (более быстрая победа → выше оценка)
- Победа противника: `-10 + depth` (более быстрое поражение → ниже оценка)

Это гарантирует:
1. Алгоритм предпочитает немедленные победы отложенным
2. Алгоритм блокирует немедленные угрозы противника
3. На пустой доске центр (индекс 4) даёт более быстрые победы в большинстве
   вариантов → получает более высокую оценку → выбирается первым

**Важно:** Алгоритм остаётся детерминированным (ADR-002) — при одинаковом
состоянии доски всегда возвращает один и тот же ход.
