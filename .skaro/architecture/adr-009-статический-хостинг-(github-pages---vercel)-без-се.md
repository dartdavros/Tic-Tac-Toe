# ADR-009: Статический хостинг (GitHub Pages / Vercel) без серверной части

**Status:** accepted
**Date:** 2026-02-28

## Context
Проект является учебным однопользовательским приложением с бюджетом $0. Вся бизнес-логика реализована на клиенте. Необходимо выбрать инфраструктуру деплоя, которая: (а) бесплатна, (б) поддерживает статические файлы (HTML + JS + CSS), (в) обеспечивает достаточную производительность для учебного проекта, (г) интегрируется с GitHub Actions для CI/CD.

## Decision
Приложение собирается командой `vite build` в набор статических файлов в директории `dist/` и деплоится на GitHub Pages или Vercel (бесплатный tier). Серверная часть отсутствует полностью. CI/CD реализован через GitHub Actions: lint → test → build → deploy. Бандл не должен превышать 100 КБ (gzip), что проверяется через `vite build --report` в CI.

## Alternatives
1. **Собственный VPS / облачный сервер (DigitalOcean, AWS EC2):** полный контроль над инфраструктурой — отклонён, поскольку требует финансовых затрат (минимум $5/месяц) и операционных усилий по администрированию; избыточен для статического SPA без серверной логики.
2. **Docker-контейнер с Nginx:** воспроизводимое окружение, простой деплой на любой платформе — отклонён, поскольку добавляет сложность (Dockerfile, образ, реестр контейнеров) без реальной пользы для статического приложения; GitHub Pages и Vercel решают ту же задачу проще и бесплатно.
3. **Netlify:** аналог Vercel с бесплатным tier и интеграцией с GitHub — не отклонён принципиально, является равнозначной альтернативой Vercel; выбор между Vercel и Netlify оставлен на усмотрение команды, оба варианта допустимы.

## Consequences
- Positive: нулевые операционные затраты; автоматический деплой при push в ветку `main`; CDN-раздача статики обеспечивает низкую задержку; нет необходимости в администрировании серверов.
- Negative: невозможен мультиплеер по сети (нет серверной части для синхронизации состояния); история игр не сохраняется между сессиями (нет базы данных); ограниченные возможности серверного мониторинга.
- Risks: при превышении лимитов бесплатного tier (например, трафик или количество деплоев) потребуется переход на платный план или смена провайдера; зависимость от стороннего сервиса (GitHub / Vercel) — при их недоступности деплой невозможен.
