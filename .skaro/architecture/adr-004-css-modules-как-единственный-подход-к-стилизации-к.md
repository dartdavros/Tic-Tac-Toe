# ADR-004: CSS Modules как единственный подход к стилизации компонентов

**Status:** accepted
**Date:** 2026-02-28

## Context
Необходимо выбрать стратегию стилизации React-компонентов, которая: (а) обеспечивает изоляцию стилей между компонентами, (б) поддерживается Vite 5 из коробки, (в) не требует дополнительных зависимостей, (г) совместима с адаптивной вёрсткой и CSS-переменными. Проект учебный, бюджет $0, команда небольшая.

## Decision
Каждый компонент имеет собственный файл `*.module.css`. Глобальные стили, сброс и CSS-переменные (токены дизайна) размещаются исключительно в `src/styles/global.css`. Смешение CSS Modules с другими подходами запрещено. Inline-стили допустимы только для динамических значений, невыразимых через CSS-переменные, с обязательным комментарием `// STYLE_EXCEPTION`.

## Alternatives
1. **styled-components / Emotion (CSS-in-JS):** удобны для динамических стилей на основе props — отклонены, поскольку добавляют значительную зависимость (~15–30 КБ), увеличивают бандл сверх лимита 100 КБ gzip, и усложняют SSR (не актуален, но создаёт ненужный технический долг).
2. **Tailwind CSS:** утилитарный подход, минимальный CSS в бандле — отклонён, поскольку требует настройки и изучения, избыточен для небольшого учебного проекта с фиксированным набором компонентов; классы в JSX снижают читаемость разметки.
3. **Глобальные CSS-файлы без модулей:** простейший подход — отклонён из-за риска конфликтов имён классов при росте проекта; отсутствует изоляция стилей между компонентами.
4. **Inline-стили для всех компонентов:** максимальная изоляция — отклонён, поскольку не поддерживает псевдоклассы (`:hover`, `:focus`), медиазапросы и CSS-переменные без дополнительных библиотек.

## Consequences
- Positive: изоляция стилей гарантирована на уровне сборщика; нет дополнительных зависимостей; Vite поддерживает CSS Modules нативно; CSS-переменные обеспечивают единую систему токенов дизайна; адаптивная вёрстка реализуется стандартными медиазапросами.
- Negative: динамические стили (например, анимации с вычисляемыми значениями) требуют либо inline-стилей с документированием исключений, либо CSS-переменных, устанавливаемых через JS.
- Risks: при значительном росте числа компонентов управление CSS-переменными в одном `global.css` может стать неудобным; потребуется разбивка на тематические файлы токенов.
