# ADR-002: Minimax (полный перебор) как алгоритм ИИ для поля 3×3

**Status:** accepted
**Date:** 2026-02-28

## Context
Приложение поддерживает режим игры против компьютера (PvAI). Необходимо выбрать алгоритм, который: (а) принимает оптимальные решения, (б) детерминирован для тестируемости, (в) выполняется достаточно быстро для синхронного вызова в браузере. Поле 3×3 имеет не более 9! = 362 880 листовых узлов дерева игры.

## Decision
Используется алгоритм minimax с полным перебором без оптимизаций. Алгоритм детерминирован: при одинаковом состоянии доски всегда возвращает один и тот же ход. Измеренное время выполнения на пустой доске (худший случай) составляет менее 5 мс, что значительно ниже установленного порога в 50 мс. Alpha-beta pruning не применяется как избыточная оптимизация для данного размера поля.

## Alternatives
1. **Minimax с alpha-beta pruning:** сокращает количество вычисляемых узлов примерно вдвое — отклонён, поскольку для поля 3×3 выигрыш в производительности незначителен, а усложнение кода и тестов не оправдано; полный перебор укладывается в < 5 мс.
2. **Эвристический алгоритм (приоритетные правила):** простая реализация на основе правил (занять центр, занять угол, заблокировать противника) — отклонён, поскольку не гарантирует оптимальной игры во всех ситуациях; minimax математически оптимален.
3. **Случайный выбор из доступных ходов:** тривиальная реализация — отклонён, поскольку делает ИИ легко обыгрываемым и не представляет интереса для пользователя.
4. **Асинхронный ИИ через Web Worker:** позволяет не блокировать основной поток — отклонён для текущего размера поля, поскольку синхронное выполнение занимает < 5 мс и не вызывает заметных задержек; при увеличении поля переход на Web Worker обязателен (зафиксировано как известный компромисс).

## Consequences
- Positive: ИИ играет оптимально и не может проиграть; детерминированность обеспечивает воспроизводимость тестов; простота реализации снижает вероятность ошибок.
- Negative: алгоритм не масштабируется на поля большего размера (например, 4×4 или 5×5) без существенной доработки или замены на alpha-beta pruning / MCTS.
- Risks: если в будущем размер поля увеличится, синхронный вызов minimax заблокирует основной поток браузера; архитектурный инвариант №7 требует обязательного создания ADR и переноса вычислений в Web Worker до слияния такого изменения.
