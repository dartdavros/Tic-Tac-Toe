# Specification: game-logic-tests

## Context
Модуль `gameLogic.ts` содержит критическую бизнес-логику, от которой зависит корректность всей игры. Согласно архитектуре, покрытие тестами должно быть ≥ 90%. Тесты должны проверять все 8 выигрышных линий, граничные случаи и корректность иммутабельности `applyMove`.

## User Scenarios
1. **Разработчик запускает тесты:** `npm test` — все тесты проходят, покрытие ≥ 90%.
2. **Разработчик вносит изменение в логику:** тесты немедленно выявляют регрессию.

## Functional Requirements
- FR-01: Тесты для `checkWinner`: победа по каждой из 3 горизонталей (индексы [0,1,2], [3,4,5], [6,7,8]).
- FR-02: Тесты для `checkWinner`: победа по каждой из 3 вертикалей (индексы [0,3,6], [1,4,7], [2,5,8]).
- FR-03: Тесты для `checkWinner`: победа по каждой из 2 диагоналей (индексы [0,4,8], [2,4,6]).
- FR-04: Тест для `checkWinner`: ничья при заполненной доске без победителя.
- FR-05: Тест для `checkWinner`: статус `playing` при незавершённой игре.
- FR-06: Тест для `isBoardFull`: возвращает `false` для пустой доски.
- FR-07: Тест для `isBoardFull`: возвращает `true` для полностью заполненной доски.
- FR-08: Тест для `isBoardFull`: возвращает `false` для частично заполненной доски.
- FR-09: Тест для `getAvailableMoves`: возвращает все 9 индексов для пустой доски.
- FR-10: Тест для `getAvailableMoves`: возвращает пустой массив для полной доски.
- FR-11: Тест для `getAvailableMoves`: возвращает корректные индексы для частично заполненной доски.
- FR-12: Тест для `applyMove`: возвращает новый `BoardState` с корректным значением.
- FR-13: Тест для `applyMove`: не мутирует исходный `BoardState`.
- FR-14: Тест для `applyMove`: выбрасывает ошибку при ходе в занятую клетку.

## Non-Functional Requirements
- NFR-01: Покрытие строк и веток для `gameLogic.ts` ≥ 90% (проверяется через `npm run test:coverage`).
- NFR-02: Каждый тест изолирован — не зависит от порядка выполнения.

## Boundaries (что НЕ входит)
- Не тестируется интеграция с редьюсером или компонентами.
- Не тестируется производительность функций логики.

## Acceptance Criteria
- [ ] Все 8 выигрышных линий покрыты отдельными тестами.
- [ ] Тест ничьей проходит корректно.
- [ ] Тест незавершённой игры проходит корректно.
- [ ] Все граничные случаи `getAvailableMoves` покрыты.
- [ ] Иммутабельность `applyMove` подтверждена тестом.
- [ ] Покрытие `gameLogic.ts` ≥ 90% по отчёту coverage.
- [ ] `npm run lint` не выдаёт ошибок для тестового файла.
