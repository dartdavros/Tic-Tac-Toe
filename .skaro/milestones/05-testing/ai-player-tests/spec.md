# Specification: ai-player-tests

## Context
Алгоритм ИИ является критической бизнес-логикой. Тесты должны подтвердить оптимальность решений, детерминированность и соответствие performance-контракту (< 50 мс). Согласно архитектуре, тест производительности является обязательным.

## User Scenarios
1. **Разработчик запускает тесты:** все тесты проходят, включая тест производительности.
2. **Разработчик изменяет алгоритм:** тесты выявляют субоптимальные решения или нарушение детерминированности.

## Functional Requirements
- FR-01: Тест детерминированности: повторный вызов `getBestMove` с одинаковыми аргументами возвращает одинаковый результат.
- FR-02: Тест победного хода: ИИ выбирает победный ход, если он доступен (для X и для O).
- FR-03: Тест блокировки: ИИ блокирует победный ход противника, если нет собственного победного хода.
- FR-04: Тест производительности: `getBestMove(EMPTY_BOARD, 'X')` выполняется менее чем за 50 мс.
- FR-05: Тест граничного случая: `getBestMove` на полной доске выбрасывает `Error('No available moves')`.
- FR-06: Тест на почти пустой доске: ИИ возвращает валидный индекс [0..8].
- FR-07: Тест для обоих символов ИИ: `getBestMove` корректно работает как для `aiPlayer = 'X'`, так и для `aiPlayer = 'O'`.

## Non-Functional Requirements
- NFR-01: Покрытие строк и веток для `aiPlayer.ts` ≥ 90%.
- NFR-02: Тест производительности использует `performance.now()` для измерения времени.

## Boundaries (что НЕ входит)
- Не тестируется интеграция с редьюсером.
- Не тестируется субъективное «качество» игры ИИ (только оптимальность в конкретных позициях).

## Acceptance Criteria
- [ ] Тест детерминированности проходит.
- [ ] ИИ всегда выбирает победный ход при его наличии.
- [ ] ИИ всегда блокирует победный ход противника.
- [ ] Тест производительности: `getBestMove(EMPTY_BOARD, 'X')` < 50 мс.
- [ ] `getBestMove` на полной доске выбрасывает ожидаемую ошибку.
- [ ] Покрытие `aiPlayer.ts` ≥ 90%.
- [ ] `npm run lint` не выдаёт ошибок.

## Open Questions
- Нужно ли тестировать конкретные позиции (например, «ИИ берёт центр на пустой доске»), или достаточно тестов оптимальности?
- Стоит ли добавить тест, что ИИ никогда не проигрывает (полная игра против случайного противника)?
