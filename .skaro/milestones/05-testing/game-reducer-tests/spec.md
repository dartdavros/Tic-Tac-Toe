# Specification: game-reducer-tests

## Context
Редьюсер является единственной точкой мутации состояния. Тесты должны покрыть все допустимые и недопустимые переходы, включая автоматический переход на экран результата и защитное поведение при невалидных действиях. Интеграционные сценарии (полная игра) покрываются здесь, а не в тестах компонентов.

## User Scenarios
1. **Разработчик тестирует полную игру PvP:** серия вызовов редьюсера с ходами приводит к победе и переходу на `result`.
2. **Разработчик тестирует защитное поведение:** невалидные действия не изменяют состояние.

## Functional Requirements
- FR-01: Тест `START_GAME` (PvP): `screen` переходит в `'game'`, доска сброшена, настройки установлены.
- FR-02: Тест `START_GAME` (PvAI, humanPlayer: 'O'): ИИ делает первый ход автоматически.
- FR-03: Тест `MAKE_MOVE`: корректный ход обновляет доску и меняет текущего игрока.
- FR-04: Тест `MAKE_MOVE`: победный ход переводит `screen` в `'result'`, `status.kind` в `'won'`.
- FR-05: Тест `MAKE_MOVE`: ход, приводящий к ничьей, переводит `screen` в `'result'`, `status.kind` в `'draw'`.
- FR-06: Тест `MAKE_MOVE` в PvAI: ход игрока + ответный ход ИИ применяются атомарно.
- FR-07: Тест невалидного `MAKE_MOVE`: ход в занятую клетку возвращает текущее состояние.
- FR-08: Тест невалидного `MAKE_MOVE`: ход при `screen !== 'game'` возвращает текущее состояние.
- FR-09: Тест невалидного `MAKE_MOVE`: ход при `status.kind !== 'playing'` возвращает текущее состояние.
- FR-10: Тест невалидного `MAKE_MOVE`: индекс вне диапазона [0..8] возвращает текущее состояние.
- FR-11: Тест `RESTART`: возвращает `INITIAL_STATE`.
- FR-12: Тест `QUIT_TO_MENU`: возвращает `INITIAL_STATE`.
- FR-13: Тест защитного поведения: при исключении внутри редьюсера (мок ошибки) возвращается текущее состояние.

## Non-Functional Requirements
- NFR-01: Тесты не используют React — только чистые вызовы функции `gameReducer`.
- NFR-02: Каждый тест изолирован и не зависит от порядка выполнения.

## Boundaries (что НЕ входит)
- Не тестируется UI-рендеринг (это smoke-тесты компонентов).
- Не тестируется производительность редьюсера отдельно (покрывается тестом `aiPlayer`).

## Acceptance Criteria
- [ ] Все 4 типа `GameAction` покрыты тестами.
- [ ] Все невалидные ходы возвращают текущее состояние без изменений.
- [ ] Победа и ничья корректно переводят `screen` в `'result'`.
- [ ] В PvAI-режиме ход игрока и ответ ИИ применяются в одном вызове редьюсера.
- [ ] `RESTART` и `QUIT_TO_MENU` возвращают `INITIAL_STATE`.
- [ ] Защитное поведение при исключении подтверждено тестом с моком.
- [ ] `npm run lint` не выдаёт ошибок.
