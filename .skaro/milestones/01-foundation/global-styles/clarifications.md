# Clarifications: global-styles

## Question 1
Нужна ли CSS-переменная для цвета подсветки победных клеток (Open Questions из спецификации)?

*Context:* Компоненты Board и Cell, согласно архитектурным инвариантам, не могут хардкодить цвета — если переменная не определена в global.css, подсветку победителя реализовать без нарушения NFR-02 невозможно.

**Options:**
- A) Добавить --color-win-highlight: #f1c40f (жёлтый) в :root
- B) Добавить --color-win-highlight: #27ae60 (переиспользовать --color-accent)
- C) Не добавлять сейчас, определить в задаче компонента Cell

**Answer:**
Добавить --color-win-highlight: #f1c40f (жёлтый) в :root

## Question 2
Нужна ли переменная --border-radius для единообразного скругления углов (второй Open Question из спецификации)?

*Context:* Без единой переменной каждый CSS-модуль будет хардкодить значение border-radius, что нарушает архитектурный инвариант №15 о запрете хардкода в *.module.css.

**Options:**
- A) Добавить --border-radius: 8px в :root
- B) Добавить --border-radius-sm: 4px и --border-radius-md: 8px (два варианта)
- C) Не добавлять, border-radius не является токеном дизайна в данном проекте

**Answer:**
Добавить --border-radius: 8px в :root

## Question 3
Какие конкретные значения уменьшаются в медиазапросе @media (max-width: 480px) — FR-07 не специфицирует их явно?

*Context:* Без конкретных значений реализация медиазапроса будет произвольной, что затруднит проверку критерия FR-08 (отображение от 320px без горизонтального скролла).

**Options:**
- A) Переопределить только --font-size-base: 14px и --font-size-large: 18px
- B) Переопределить --font-size-base: 14px, --font-size-large: 18px и --cell-size-mobile: 72px
- C) Переопределить все размерные переменные: шрифты, cell-size и добавить --spacing-mobile

**Answer:**
Переопределить только --font-size-base: 14px и --font-size-large: 18px

## Question 4
Должен ли global.css определять переменные отступов (spacing tokens), которые будут использоваться в CSS-модулях компонентов?

*Context:* Спецификация упоминает 'уменьшенные отступы' в FR-07, но не определяет переменных для отступов в FR-02–FR-04, что означает либо пропуск токенов, либо хардкод в модулях.

**Options:**
- A) Добавить --spacing-sm: 8px, --spacing-md: 16px, --spacing-lg: 32px в :root
- B) Добавить только --spacing-base: 16px как единственный токен отступа
- C) Не добавлять, отступы задаются хардкодом в CSS-модулях (исключение из инварианта)

**Answer:**
Добавить только --spacing-base: 16px как единственный токен отступа

## Question 5
Нужна ли переменная --cell-size-desktop в медиазапросе, или она применяется только вне медиазапросов (в :root по умолчанию)?

*Context:* FR-03 определяет обе переменные --cell-size-mobile и --cell-size-desktop, но не указывает, как они соотносятся с медиазапросами — это влияет на то, какую переменную использует компонент Cell.

**Options:**
- A) --cell-size-desktop применяется по умолчанию в :root, --cell-size-mobile переопределяет её через @media (max-width: 480px) как значение --cell-size-desktop
- B) Компонент Cell сам выбирает нужную переменную через медиазапрос в своём CSS-модуле
- C) Добавить единую --cell-size, которая переопределяется в медиазапросе (80px / 120px)

**Answer:**
--cell-size-desktop применяется по умолчанию в :root, --cell-size-mobile переопределяет её через @media (max-width: 480px) как значение --cell-size-desktop
