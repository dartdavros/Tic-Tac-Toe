## Почему такой подход

### Переопределение `--cell-size-desktop` в медиазапросе вместо отдельной переменной
Согласно уточнению Q5 (ответ A): компонент `Cell` будет использовать единственную
переменную `--cell-size-desktop` как основной размер клетки. В медиазапросе
`--cell-size-desktop` переопределяется значением `80px` (равным `--cell-size-mobile`),
что позволяет компонентам не менять ссылки при адаптации под мобильные экраны.
Переменная `--cell-size-mobile` остаётся в `:root` как справочное значение
(документирует намерение) и не используется напрямую в медиазапросе.

### Единственный токен отступа `--spacing-base`
Согласно уточнению Q4 (ответ B): добавлен только `--spacing-base: 16px`.
Дополнительные токены (`--spacing-sm`, `--spacing-lg`) не добавлялись —
они выходят за рамки текущей задачи и могут быть добавлены в отдельном ADR.

### `--color-win-highlight: #f1c40f`
Согласно уточнению Q1 (ответ A): жёлтый цвет для подсветки победных клеток.
Переиспользование `--color-accent` (зелёный) отклонено — семантически
победная подсветка и акцентный цвет интерфейса — разные сущности.

### `--border-radius: 8px`
Согласно уточнению Q2 (ответ A): единственная переменная скругления.
Два варианта (`--border-radius-sm` / `--border-radius-md`) не добавлялись —
избыточно для текущего масштаба проекта.

### Порядок импорта в `main.tsx`
Импорт `global.css` размещён после импортов React и компонентов, но перед
вызовом `createRoot`. Это стандартный порядок для Vite: CSS-файлы
обрабатываются сборщиком независимо от порядка импортов в JS,
однако явный порядок улучшает читаемость.

### Отсутствие компонентно-специфичных стилей
Файл содержит исключительно: блок `:root` с переменными, CSS-сброс для
`*, *::before, *::after` и `body`, медиазапрос с переопределением переменных.
Никаких классов, идентификаторов или селекторов компонентов — соответствует NFR-01.

## Риски и ограничения

- **Опечатка в имени переменной** — все 16 имён сверены с таблицей спецификации
  вручную. Риск минимален, но при рефакторинге рекомендуется поиск по имени
  переменной во всём проекте.
- **`--cell-size-mobile` как справочное значение** — переменная определена в `:root`,
  но не используется в медиазапросе напрямую (там хардкодится `80px`).
  Это сделано намеренно: значение `80px` совпадает с `--cell-size-mobile`,
  и дублирование допустимо внутри одного файла токенов. Альтернатива —
  использовать `var(--cell-size-mobile)` в медиазапросе — невозможна, так как
  CSS не поддерживает `var()` внутри `@media` условий (только в объявлениях свойств).
- **Системные шрифты** — `'Segoe UI', system-ui, sans-serif` не требуют
  подключения через `@import`. На системах без Segoe UI (macOS, Linux)
  будет использован `system-ui` или `sans-serif` — это ожидаемое поведение.

## Соответствие архитектурным инвариантам

- [x] **Инвариант №14** (CSS Modules — единственный механизм стилизации компонентов) —
  соблюдён: `global.css` не является CSS-модулем и не содержит компонентных стилей;
  он подключается глобально как точка определения токенов
- [x] **Инвариант №15** (все токены дизайна — только в `global.css`) —
  соблюдён: все 16 переменных определены в `:root` файла `global.css`;
  хардкод значений в `*.module.css` теперь запрещён и обеспечен единым источником
- [x] **NFR-01** (нет компонентно-специфичных стилей) — соблюдён: файл содержит
  только сброс и переменные
- [x] **NFR-02** (все значения через CSS-переменные) — соблюдён: `body` использует
  исключительно `var(--...)` ссылки; единственное исключение — значение `0`
  для `margin` и `padding` (не является токеном дизайна)

## Как проверить

1. `npm run build` — сборка должна завершиться без ошибок
2. Открыть приложение в браузере, DevTools → Elements → выбрать `:root` →
   вкладка Computed → убедиться, что видны все 16 переменных, включая
   `--color-win-highlight`, `--border-radius`, `--spacing-base`
3. В DevTools → Responsive Mode установить ширину `375px` (мобильный) →
   Computed → убедиться: `--font-size-base: 14px`, `--font-size-large: 18px`,
   `--cell-size-desktop: 80px`
4. Установить ширину `1024px` (десктоп) → Computed → убедиться:
   `--font-size-base: 16px`, `--font-size-large: 24px`, `--cell-size-desktop: 120px`
5. Установить ширину `320px` → убедиться в отсутствии горизонтального скролла
6. Проверить `src/main.tsx`: строка `import './styles/global.css'` присутствует
