# Specification: game-reducer

## Context
Редьюсер является единственной точкой мутации игрового состояния (архитектурный инвариант №1). Он обрабатывает все действия `GameAction`, валидирует ходы, вызывает логику игры и ИИ, управляет переходами между экранами. Согласно ADR-007, редьюсер защищён `try/catch` и никогда не выбрасывает исключений.

## User Scenarios
1. **Игрок делает ход в свободную клетку (PvP):** состояние обновляется, ход переходит к следующему игроку.
2. **Игрок делает ход, завершающий игру победой:** редьюсер автоматически переводит `screen` в `'result'`, `status` в `{ kind: 'won', ... }`.
3. **Игрок делает ход в PvAI-режиме:** редьюсер применяет ход игрока, вычисляет ход ИИ синхронно, применяет его — всё в одной транзакции.
4. **Игрок кликает на занятую клетку:** редьюсер возвращает текущее состояние без изменений.
5. **Игрок нажимает «Начать заново»:** редьюсер сбрасывает доску, статус и настройки, переводит `screen` в `'menu'` (возврат к `INITIAL_STATE`).

## Functional Requirements
- FR-01: Экспортирована константа `INITIAL_STATE: GameState` с `screen: 'menu'`, пустой доской, `status: { kind: 'playing', currentPlayer: 'X' }`, `settings: { mode: 'pvp' }`.
- FR-02: Экспортирована константа `EMPTY_BOARD: BoardState` — кортеж из 9 `null`.
- FR-03: Обработка `START_GAME`: сброс доски, установка настроек из payload, `screen: 'game'`. Если режим `pvai` и ИИ ходит первым (`humanPlayer === 'O'`), немедленно вычисляется и применяется первый ход ИИ. После этого хода обязательно вызывается `checkWinner` для единообразия с FR-05/FR-06. Если `getBestMove` возвращает `null` или `-1` — логировать ошибку через `logEvent` и оставить доску без хода ИИ.
- FR-04: Обработка `MAKE_MOVE`: валидация (`screen === 'game'`, `status.kind === 'playing'`, `index` в `[0..8]`, клетка свободна). При невалидном ходе — возврат текущего состояния без изменений.
- FR-05: После применения хода игрока вызывается `checkWinner`. Если игра завершена — `screen: 'result'`, соответствующий `status`.
- FR-06: Если игра продолжается и режим `pvai` и сейчас ход ИИ (определяется через `isAiTurn`) — синхронно вызывается `getBestMove`, применяется ход ИИ, снова проверяется победитель. Если `getBestMove` возвращает `null` или `-1` — логировать ошибку через `logEvent('reducer_error', { action, error })` и вернуть текущее состояние с уже применённым ходом игрока, без хода ИИ.
- FR-07: Обработка `RESTART`: возврат к `INITIAL_STATE` (включая `settings: { mode: 'pvp' }` и `screen: 'menu'`).
- FR-08: Обработка `QUIT_TO_MENU`: возврат к `INITIAL_STATE`.
- FR-09: Весь код обработки действий обёрнут в `try/catch`. При исключении — логирование через `logEvent('reducer_error', { action, error })`, где `action` — полный объект `GameAction`, и возврат текущего состояния. `logEvent` вызывается **исключительно** в блоке `catch`; успешные ходы не логируются.
- FR-10: Недопустимые переходы игнорируются: `MAKE_MOVE` при `screen !== 'game'`, ход в занятую клетку, ход при завершённой партии.
- FR-11: Переход `screen: 'result'` происходит только внутри редьюсера при обработке `MAKE_MOVE` (архитектурный инвариант №5).
- FR-12: Экспортирована чистая функция `isAiTurn(status: GameStatus, settings: GameSettings): boolean`. Функция возвращает `true`, если режим `pvai` и `status.kind === 'playing'` и `status.currentPlayer` не совпадает с `humanPlayer`. Функция используется внутри редьюсера (FR-06) и доступна для прямого импорта в тестах.

## Non-Functional Requirements
- NFR-01: Редьюсер является чистой функцией (детерминирован при одинаковых входных данных, нет мутации состояния).
- NFR-02: Редьюсер не выбрасывает исключений при любых входных данных.
- NFR-03: Обработка `MAKE_MOVE` в PvAI-режиме (включая вызов minimax) выполняется менее чем за 50 мс.

## Boundaries (что НЕ входит)
- Не реализуется логирование (используется готовый `logger` из отдельной задачи).
- Не реализуется UI — только чистая логика редьюсера.
- Не реализуется персистентность состояния.

## Acceptance Criteria
- [ ] `START_GAME` корректно инициализирует игру в обоих режимах (PvP и PvAI).
- [ ] В PvAI-режиме, если ИИ ходит первым, его ход применяется сразу при `START_GAME`.
- [ ] После первого хода ИИ при `START_GAME` вызывается `checkWinner` (результат отражается в состоянии).
- [ ] Если `getBestMove` возвращает `null` или `-1` при `START_GAME`, вызывается `logEvent` и доска остаётся без хода ИИ.
- [ ] `MAKE_MOVE` с невалидным индексом возвращает текущее состояние.
- [ ] `MAKE_MOVE` в занятую клетку возвращает текущее состояние.
- [ ] `MAKE_MOVE` при `screen !== 'game'` возвращает текущее состояние.
- [ ] Победа корректно переводит `screen` в `'result'` с `status.kind === 'won'`.
- [ ] Ничья корректно переводит `screen` в `'result'` с `status.kind === 'draw'`.
- [ ] В PvAI-режиме ход игрока и ответ ИИ применяются атомарно (один вызов редьюсера).
- [ ] Если `getBestMove` возвращает `null` или `-1` в `MAKE_MOVE`, вызывается `logEvent` и возвращается состояние с уже применённым ходом игрока.
- [ ] `RESTART` возвращает `INITIAL_STATE` полностью, включая `settings: { mode: 'pvp' }` и `screen: 'menu'`.
- [ ] `QUIT_TO_MENU` возвращает `INITIAL_STATE`.
- [ ] При исключении внутри редьюсера возвращается текущее состояние (тест с мок-ошибкой).
- [ ] При исключении внутри редьюсера `logEvent` вызывается с полным объектом `action` (не только `action.type`).
- [ ] `logEvent` **не** вызывается при успешном выполнении любого действия.
- [ ] `isAiTurn` экспортируется и доступна для прямого импорта.
- [ ] `isAiTurn` возвращает `false` для режима `pvp` при любом `status`.
- [ ] `isAiTurn` возвращает `true`, когда текущий игрок — ИИ в режиме `pvai`.
- [ ] `isAiTurn` возвращает `false`, когда текущий игрок — человек в режиме `pvai`.
- [ ] `isAiTurn` возвращает `false`, если `status.kind !== 'playing'`.
- [ ] `npm run lint` не выдаёт ошибок.
