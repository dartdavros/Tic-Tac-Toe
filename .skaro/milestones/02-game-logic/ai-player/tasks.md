## Стадия 1: Реализация модуля ИИ (aiPlayer) и его тестов

**Цель:** Реализовать алгоритм minimax в `src/logic/aiPlayer.ts` и полный набор unit-тестов в `src/logic/aiPlayer.test.ts`, удовлетворяющих всем функциональным требованиям спецификации, критериям приёмки и требованию покрытия ≥ 90%.

**Зависит от:** нет (все зависимости — `src/types/index.ts`, `src/logic/gameLogic.ts` — уже существуют в проекте)

**Параллельность:** не применимо (единственная стадия)

**Входные данные:**
- `src/types/index.ts` — типы `BoardState`, `Player`, `CellValue`
- `src/logic/gameLogic.ts` — функции `checkWinner`, `isBoardFull`, `getAvailableMoves`, `applyMove`
- Спецификация `ai-player` (FR-01 — FR-10, NFR-01 — NFR-03)
- Уточнения (Clarifications Q1–Q6)
- Архитектурные инварианты №3, №4, №7

**Выходные данные:**
- `src/logic/aiPlayer.ts` — реализация `getBestMove` и `minimax`
- `src/logic/aiPlayer.test.ts` — полный набор unit-тестов

**Детали реализации `src/logic/aiPlayer.ts`:**

1. Импорт типов из `src/types/index.ts` (инвариант: только оттуда).
2. Импорт `checkWinner`, `isBoardFull`, `getAvailableMoves`, `applyMove` из `src/logic/gameLogic.ts`.
3. Экспортируемая функция `minimax(board, aiPlayer, currentPlayer, isMaximizing): number`:
   - Терминальные условия: победа ИИ → +10, победа противника → −10, ничья → 0.
   - Рекурсивный перебор всех доступных ходов.
   - Оценка фиксирована, без поправки на глубину (FR-05, Clarification Q2).
   - Длина ≤ 40 строк (Constitution).
4. Экспортируемая функция `getBestMove(board, aiPlayer): number`:
   - Проверка победителя первой → `Error('Game is already over')` (FR-10, Clarification Q6).
   - Проверка заполненности → `Error('No available moves')` (FR-04).
   - Перебор доступных ходов, вызов `minimax`, выбор первого хода с максимальным счётом (FR-03, Clarification Q1).
   - Нет валидации структуры `board` (FR-09, Clarification Q4).
   - Длина ≤ 40 строк (Constitution).
5. Нет глобального состояния, нет побочных эффектов (NFR-03, инвариант №3).

**Детали реализации `src/logic/aiPlayer.test.ts`:**

Тесты покрывают все критерии приёмки:
- ИИ выбирает победный ход (если доступен).
- ИИ блокирует победный ход противника (если нет своего победного).
- Детерминированность: повторные вызовы на одной доске → одинаковый результат.
- При равнозначных ходах выбирается первый по индексу.
- Пустая доска → детерминированный ход (центр, индекс 4).
- Полная доска → `Error('No available moves')`.
- Доска с победителем → `Error('Game is already over')`.
- Порядок проверок: победитель проверяется раньше заполненности.
- `minimax` возвращает +10 / −10 / 0 без поправки на глубину.
- `minimax` доступен как именованный экспорт.
- Тест производительности: `getBestMove(EMPTY_BOARD, 'X')` < 50 мс (NFR-01).

- [ ] `getBestMove` выбрасывает `Error('Game is already over')` при наличии победителя на доске
- [ ] `getBestMove` выбрасывает `Error('No available moves')` при полностью заполненной доске без победителя

- [ ] ИИ всегда выбирает победный ход, если он доступен (проверено тестом)
- [ ] ИИ всегда блокирует победный ход противника при отсутствии своего победного хода (проверено тестом)

**Риски:**
- `getAvailableMoves` или `applyMove` в `gameLogic.ts` могут иметь сигнатуру, отличающуюся от ожидаемой — необходимо сверить перед реализацией и адаптировать вызовы.
- `checkWinner` может возвращать тип, отличный от ожидаемого (`Player | null` vs объект `GameStatus`) — необходимо проверить и использовать корректно.
- Если `applyMove` мутирует переданный массив вместо возврата нового — minimax будет работать некорректно; необходимо убедиться в иммутабельности или создавать копию вручную.
