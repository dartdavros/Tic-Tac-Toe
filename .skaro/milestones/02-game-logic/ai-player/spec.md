# Specification: ai-player

## Context
Алгоритм ИИ обеспечивает режим игры против компьютера (PvAI). Согласно ADR-002, используется minimax с полным перебором без оптимизаций — для поля 3×3 это тривиально по производительности (< 5 мс). Алгоритм детерминирован: при нескольких равнозначных ходах с одинаковым minimax-счётом всегда выбирается первый по индексу ход (ADR-002). Согласно архитектурному инварианту №4, `getBestMove` вызывается только из `gameReducer`.

## User Scenarios
1. **ИИ может выиграть за один ход:** `getBestMove` возвращает индекс победного хода.
2. **Противник может выиграть за один ход:** `getBestMove` возвращает индекс блокирующего хода.
3. **Пустая доска (первый ход):** `getBestMove` возвращает детерминированный оптимальный ход (например, центр — индекс 4).
4. **Полная доска:** `getBestMove` выбрасывает `Error('No available moves')`.
5. **Игра уже завершена (на доске есть победитель):** `getBestMove` выбрасывает `Error('Game is already over')`.

## Functional Requirements
- FR-01: Реализована функция `getBestMove(board: BoardState, aiPlayer: Player): number`. Возвращает индекс оптимального хода [0..8]. Типы `BoardState` и `Player` импортируются из `src/types/game.ts`.
- FR-02: Алгоритм — minimax с полным перебором. ИИ максимизирует свой счёт и минимизирует счёт противника.
- FR-03: Функция детерминирована: при одинаковом `board` и `aiPlayer` всегда возвращает одинаковый индекс. При нескольких равнозначных ходах с одинаковым minimax-счётом выбирается первый по индексу ход.
- FR-04: При полностью заполненной доске (`isBoardFull` возвращает `true`) функция выбрасывает `Error('No available moves')`.
- FR-05: Реализована вспомогательная функция `minimax(board: BoardState, aiPlayer: Player, currentPlayer: Player, isMaximizing: boolean): number`. Возвращает оценку позиции: +10 (победа ИИ), -10 (победа противника), 0 (ничья). Оценка фиксирована и не учитывает глубину дерева.
- FR-06: Функция `getBestMove` экспортируется из `src/logic/aiPlayer.ts` как именованный экспорт.
- FR-07: Длина каждой функции не превышает 40 строк.
- FR-08: Функция `minimax` экспортируется из `src/logic/aiPlayer.ts` как именованный экспорт для упрощения unit-тестирования.
- FR-09: Функция `getBestMove` не выполняет валидацию корректности переданного `board` (длины массива, допустимых значений ячеек) — корректность данных гарантируется вызывающим `gameReducer` согласно ADR, инвариант №4.
- FR-10: Если на доске уже есть победитель (игра завершена), `getBestMove` выбрасывает `Error('Game is already over')`. Проверка победителя выполняется до проверки заполненности доски.

## Non-Functional Requirements
- NFR-01: `getBestMove` выполняется менее чем за 50 мс на любом валидном состоянии доски (проверяется автоматическим тестом).
- NFR-02: Покрытие тестами модуля `aiPlayer.ts` ≥ 90%.
- NFR-03: Функция не использует глобальное состояние и не имеет побочных эффектов.

## Boundaries (что НЕ входит)
- Не реализуется alpha-beta pruning (согласно ADR-002, избыточно для поля 3×3).
- Не реализуется случайный выбор среди равнозначных ходов (алгоритм строго детерминирован: выбирается первый по индексу ход).
- Не реализуется Web Worker (синхронный вызов достаточен для поля 3×3).
- Не реализуется оценка с учётом глубины дерева (используются фиксированные оценки +10/-10/0).
- Не реализуется валидация входных данных `board` внутри `getBestMove` и `minimax`.

## Acceptance Criteria
- [ ] ИИ всегда выбирает победный ход, если он доступен.
- [ ] ИИ всегда блокирует победный ход противника, если нет собственного победного хода.
- [ ] На пустой доске `getBestMove` возвращает один и тот же индекс при повторных вызовах (детерминированность).
- [ ] При нескольких равнозначных ходах с одинаковым minimax-счётом `getBestMove` возвращает ход с наименьшим индексом.
- [ ] `getBestMove` на полной доске выбрасывает `Error('No available moves')`.
- [ ] `getBestMove` на доске с уже имеющимся победителем выбрасывает `Error('Game is already over')`.
- [ ] Проверка победителя в `getBestMove` выполняется раньше проверки заполненности доски.
- [ ] `minimax` возвращает +10 при победе ИИ, -10 при победе противника, 0 при ничьей — без поправки на глубину.
- [ ] `minimax` экспортируется как именованный экспорт и доступен для прямого импорта в тестах.
- [ ] `getBestMove` экспортируется как именованный экспорт из `src/logic/aiPlayer.ts`.
- [ ] Типы `BoardState` и `Player` импортируются исключительно из `src/types/game.ts`.
- [ ] Тест производительности: `getBestMove(EMPTY_BOARD, 'X')` выполняется менее чем за 50 мс.
- [ ] Покрытие тестами ≥ 90%.
- [ ] `npm run lint` не выдаёт ошибок.
