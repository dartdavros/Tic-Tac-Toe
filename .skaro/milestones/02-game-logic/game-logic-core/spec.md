# Specification: game-logic-core

## Context
Чистые функции игровой логики являются фундаментом всего приложения. Они используются редьюсером, алгоритмом ИИ и тестами. Изоляция логики от React позволяет тестировать её независимо и гарантирует корректность игровых правил. Согласно архитектурному инварианту №3, функции в `logic/` не импортируют ничего из `components/` и не используют React API.

## User Scenarios
1. **Игрок выигрывает по горизонтали:** три одинаковых символа в ряду — `checkWinner` возвращает `{ kind: 'won', winner: 'X', winLine: [0, 1, 2] }`.
2. **Доска заполнена без победителя:** `isBoardFull` возвращает `true`, `checkWinner` возвращает `{ kind: 'draw' }`.
3. **Игра продолжается:** есть свободные клетки и нет победителя — `checkWinner` возвращает `{ kind: 'playing', currentPlayer: ... }`.

## Functional Requirements
- FR-01: Реализована функция `checkWinner(board: BoardState, currentPlayer: Player): GameStatus`. Проверяет все 8 выигрышных линий (3 горизонтали, 3 вертикали, 2 диагонали) **для обоих игроков (`'X'` и `'O'`)**, независимо от значения `currentPlayer`. Параметр `currentPlayer` используется исключительно для формирования статуса `{ kind: 'playing', currentPlayer }`. Приоритет статусов: сначала проверяются выигрышные линии (победа имеет приоритет над ничьей), затем заполненность доски. Возвращает:
  - `{ kind: 'won', winner, winLine }` — если любой из игроков образовал выигрышную линию; `winner` — это игрок, чьи символы образовали линию;
  - `{ kind: 'draw' }` — если доска заполнена и победителя нет;
  - `{ kind: 'playing', currentPlayer }` — если игра продолжается.
- FR-02: Константа `WIN_LINES: readonly number[][]` содержит все 8 выигрышных комбинаций индексов.
- FR-03: Реализована функция `isBoardFull(board: BoardState): boolean`. Возвращает `true`, если все 9 клеток заняты.
- FR-04: Реализована функция `getAvailableMoves(board: BoardState): number[]`. Возвращает массив индексов пустых клеток **в строго возрастающем порядке (от 0 до 8)**, что гарантирует детерминизм алгоритма minimax. Для пустой доски возвращает `[0,1,2,3,4,5,6,7,8]`. Для полной доски возвращает `[]`.
- FR-05: Реализована функция `applyMove(board: BoardState, index: number, player: Player): BoardState`. Возвращает новый `BoardState` с ходом игрока. Не мутирует исходный массив. Условия выброса `Error`:
  - если индекс невалиден (< 0 или > 8) — выбрасывает `Error` с сообщением `'Invalid move: index out of range'`;
  - если клетка уже занята — выбрасывает `Error` с сообщением `'Invalid move: cell is already occupied'`.
  - Тесты проверяют точный текст сообщений об ошибках.
- FR-06: Все функции экспортируются из `src/logic/gameLogic.ts`.
- FR-07: Ни одна функция не превышает 40 строк (согласно стандартам кодирования).
- FR-08: Реализована и экспортирована функция `getNextPlayer(current: Player): Player` из `src/logic/gameLogic.ts`. Возвращает `'O'` если передан `'X'`, и `'X'` если передан `'O'`. Используется редьюсером и другими модулями вместо inline-логики.

## Non-Functional Requirements
- NFR-01: Покрытие тестами модуля `gameLogic.ts` ≥ 90% (строки и ветки).
- NFR-02: Функции являются чистыми: нет побочных эффектов, нет мутации входных данных.
- NFR-03: `applyMove` создаёт новый кортеж `BoardState`, не изменяя исходный.

## Boundaries (что НЕ входит)
- Не реализуется алгоритм ИИ (отдельная задача).
- Не реализуется редьюсер (отдельная задача).
- Не реализуется валидация хода (выполняется в редьюсере).

## Acceptance Criteria
- [ ] `checkWinner` корректно определяет победу по всем 8 линиям.
- [ ] `checkWinner` проверяет победителя среди обоих игроков (`'X'` и `'O'`), а не только для переданного `currentPlayer`.
- [ ] `checkWinner` возвращает `winner` равным игроку, чьи символы образовали выигрышную линию.
- [ ] `checkWinner` возвращает `{ kind: 'won', ... }` если доска одновременно полна и содержит выигрышную линию (победа имеет приоритет над ничьей).
- [ ] `checkWinner` возвращает `draw` при заполненной доске без победителя.
- [ ] `checkWinner` возвращает `playing` при незавершённой игре; поле `currentPlayer` совпадает с переданным аргументом.
- [ ] `isBoardFull` возвращает `true` только при всех 9 занятых клетках.
- [ ] `getAvailableMoves` возвращает корректный список для пустой, частично заполненной и полной доски.
- [ ] `getAvailableMoves` всегда возвращает индексы в строго возрастающем порядке (0→8).
- [ ] `applyMove` не мутирует исходный `BoardState`.
- [ ] `applyMove` выбрасывает `Error` с сообщением `'Invalid move: cell is already occupied'` при попытке хода в занятую клетку.
- [ ] `applyMove` выбрасывает `Error` с сообщением `'Invalid move: index out of range'` при индексе < 0 или > 8.
- [ ] `getNextPlayer('X')` возвращает `'O'`; `getNextPlayer('O')` возвращает `'X'`.
- [ ] Покрытие тестами ≥ 90% (проверяется через `npm run test:coverage`).
- [ ] `npm run lint` не выдаёт ошибок для файлов `gameLogic.ts` и `gameLogic.test.ts`.
