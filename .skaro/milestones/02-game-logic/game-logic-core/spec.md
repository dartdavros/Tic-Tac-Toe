# Specification: game-logic-core

## Context
Чистые функции игровой логики являются фундаментом всего приложения. Они используются редьюсером, алгоритмом ИИ и тестами. Изоляция логики от React позволяет тестировать её независимо и гарантирует корректность игровых правил. Согласно архитектурному инварианту №3, функции в `logic/` не импортируют ничего из `components/` и не используют React API.

## User Scenarios
1. **Игрок выигрывает по горизонтали:** три одинаковых символа в ряду — `checkWinner` возвращает `{ kind: 'won', winner: 'X', winLine: [0, 1, 2] }`.
2. **Доска заполнена без победителя:** `isBoardFull` возвращает `true`, `checkWinner` возвращает `{ kind: 'draw' }`.
3. **Игра продолжается:** есть свободные клетки и нет победителя — `checkWinner` возвращает `{ kind: 'playing', currentPlayer: ... }`.

## Functional Requirements
- FR-01: Реализована функция `checkWinner(board: BoardState, currentPlayer: Player): GameStatus`. Проверяет все 8 выигрышных линий (3 горизонтали, 3 вертикали, 2 диагонали). Возвращает `{ kind: 'won', winner, winLine }` при победе, `{ kind: 'draw' }` при заполненной доске без победителя, `{ kind: 'playing', currentPlayer }` иначе.
- FR-02: Константа `WIN_LINES: readonly number[][]` содержит все 8 выигрышных комбинаций индексов.
- FR-03: Реализована функция `isBoardFull(board: BoardState): boolean`. Возвращает `true`, если все 9 клеток заняты.
- FR-04: Реализована функция `getAvailableMoves(board: BoardState): number[]`. Возвращает массив индексов пустых клеток. Для пустой доски возвращает `[0,1,2,3,4,5,6,7,8]`. Для полной доски возвращает `[]`.
- FR-05: Реализована функция `applyMove(board: BoardState, index: number, player: Player): BoardState`. Возвращает новый `BoardState` с ходом игрока. Не мутирует исходный массив. Если клетка занята или индекс невалиден — выбрасывает `Error`.
- FR-06: Все функции экспортируются из `src/logic/gameLogic.ts`.
- FR-07: Ни одна функция не превышает 40 строк (согласно стандартам кодирования).

## Non-Functional Requirements
- NFR-01: Покрытие тестами модуля `gameLogic.ts` ≥ 90% (строки и ветки).
- NFR-02: Функции являются чистыми: нет побочных эффектов, нет мутации входных данных.
- NFR-03: `applyMove` создаёт новый кортеж `BoardState`, не изменяя исходный.

## Boundaries (что НЕ входит)
- Не реализуется алгоритм ИИ (отдельная задача).
- Не реализуется редьюсер (отдельная задача).
- Не реализуется валидация хода (выполняется в редьюсере).

## Acceptance Criteria
- [ ] `checkWinner` корректно определяет победу по всем 8 линиям.
- [ ] `checkWinner` возвращает `draw` при заполненной доске без победителя.
- [ ] `checkWinner` возвращает `playing` при незавершённой игре.
- [ ] `isBoardFull` возвращает `true` только при всех 9 занятых клетках.
- [ ] `getAvailableMoves` возвращает корректный список для пустой, частично заполненной и полной доски.
- [ ] `applyMove` не мутирует исходный `BoardState`.
- [ ] `applyMove` выбрасывает ошибку при попытке хода в занятую клетку.
- [ ] Покрытие тестами ≥ 90% (проверяется через `npm run test:coverage`).
- [ ] `npm run lint` не выдаёт ошибок для файлов `gameLogic.ts` и `gameLogic.test.ts`.

## Open Questions
- Должна ли `checkWinner` принимать `currentPlayer` для определения следующего хода в статусе `playing`, или следующий игрок вычисляется в редьюсере?
- Нужна ли отдельная функция `getNextPlayer(current: Player): Player` или это inline-логика в редьюсере?
