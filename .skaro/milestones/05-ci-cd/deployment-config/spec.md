# Specification: deployment-config

## Context
Приложение деплоится как набор статических файлов на GitHub Pages или Vercel (бесплатный tier). Согласно ADR-009, серверная часть отсутствует. Необходимо настроить корректный `base` URL в Vite для GitHub Pages (если репозиторий не является корневым) и добавить шаг деплоя в CI.

## User Scenarios
1. **Разработчик пушит в main:** через 2-3 минуты приложение доступно по публичному URL.
2. **Пользователь открывает приложение по URL:** страница загружается корректно, все ресурсы (JS, CSS) доступны.
3. **Пользователь обновляет страницу:** приложение загружается снова с начального экрана (меню).

## Functional Requirements
- FR-01: `vite.config.ts` содержит корректный `base` для выбранного хостинга (например, `'/repo-name/'` для GitHub Pages или `'/'` для Vercel).
- FR-02: Для GitHub Pages: добавлен шаг деплоя в `.github/workflows/ci.yml` с использованием `actions/deploy-pages` или `peaceiris/actions-gh-pages`.
- FR-03: Для Vercel: создан файл `vercel.json` с настройками (если необходимо) или настроен через Vercel Dashboard.
- FR-04: `npm run build` создаёт `dist/` с корректными относительными путями к ресурсам.
- FR-05: Приложение доступно по публичному URL после деплоя.
- FR-06: Размер бандла не превышает 100 КБ (gzip) — проверяется через `vite build --report`.
- FR-07: В `README.md` добавлена ссылка на задеплоенное приложение.

## Non-Functional Requirements
- NFR-01: Время загрузки приложения на соединении 3G < 1 секунды (первый рендер).
- NFR-02: Деплой происходит автоматически без ручных действий после push в main.

## Boundaries (что НЕ входит)
- Не настраивается кастомный домен.
- Не настраивается HTTPS (предоставляется хостингом автоматически).
- Не настраивается CDN (используется встроенный CDN хостинга).
- Не настраивается мониторинг uptime.

## Acceptance Criteria
- [ ] `npm run build` завершается без ошибок, `dist/` создан.
- [ ] Размер бандла ≤ 100 КБ (gzip) по отчёту `vite build --report`.
- [ ] Приложение доступно по публичному URL.
- [ ] Все ресурсы (JS, CSS) загружаются без 404-ошибок.
- [ ] Обновление страницы не вызывает ошибок.
- [ ] Ссылка на приложение добавлена в `README.md`.

## Open Questions
