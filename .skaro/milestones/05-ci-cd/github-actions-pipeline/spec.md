# Specification: github-actions-pipeline

## Context
CI/CD пайплайн обеспечивает автоматическую проверку качества кода и сборки при каждом push и pull request. Согласно архитектуре, пайплайн включает lint → test (с coverage) → build → deploy. Без CI невозможно гарантировать, что изменения не нарушают тесты или сборку.

## User Scenarios
1. **Разработчик создаёт pull request:** GitHub Actions автоматически запускает lint, тесты и сборку — результат виден в PR.
2. **Разработчик пушит в main:** пайплайн проходит все проверки и деплоит приложение.
3. **Разработчик пушит код с ошибкой линтера:** пайплайн завершается с ошибкой на шаге lint, деплой не происходит.

## Functional Requirements
- FR-01: Создан файл `.github/workflows/ci.yml`.
- FR-02: Пайплайн запускается на `push` и `pull_request` для всех веток.
- FR-03: Шаг `install`: `npm ci` для детерминированной установки зависимостей.
- FR-04: Шаг `lint`: `npm run lint` — завершается с ошибкой при наличии ESLint-предупреждений.
- FR-05: Шаг `format`: `npm run format:check` — завершается с ошибкой при несоответствии форматированию.
- FR-06: Шаг `test`: `npm run test:coverage` — завершается с ошибкой при покрытии ниже порогов (60% общее, 90% для `logic/`).
- FR-07: Шаг `build`: `npm run build` — завершается с ошибкой при ошибках TypeScript или превышении размера бандла.
- FR-08: Шаг `deploy` выполняется только на ветке `main` после успешного прохождения всех предыдущих шагов.
- FR-09: Используется Node.js версии 20 (LTS).
- FR-10: Результаты coverage кэшируются или публикуются как артефакт для просмотра.

## Non-Functional Requirements
- NFR-01: Полный пайплайн (без деплоя) выполняется менее чем за 3 минуты.
- NFR-02: Зависимости кэшируются между запусками (`actions/cache` для `node_modules`).

## Boundaries (что НЕ входит)
- Не настраивается матрица тестирования на нескольких версиях Node.js.
- Не настраивается тестирование в нескольких браузерах (Vitest использует jsdom).
- Не настраивается уведомление в Slack/Telegram при падении пайплайна.

## Acceptance Criteria
- [ ] Файл `.github/workflows/ci.yml` создан и валиден (проверяется GitHub).
- [ ] Пайплайн успешно выполняется на чистом репозитории.
- [ ] Пайплайн завершается с ошибкой при наличии ESLint-ошибок (проверяется вручную).
- [ ] Пайплайн завершается с ошибкой при покрытии ниже порогов.
- [ ] Шаг `deploy` выполняется только на ветке `main`.
- [ ] Зависимости кэшируются между запусками.
