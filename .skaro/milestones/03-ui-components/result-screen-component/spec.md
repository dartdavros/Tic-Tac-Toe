# Specification: result-screen-component

## Context
Компонент `ResultScreen` отображается после завершения партии. Он показывает итог игры и предоставляет пользователю возможность начать новую партию или вернуться в меню. Кнопки используют стандартный `<button>` с понятным текстом (согласно ADR-010). Компонент полностью заменяет `Board` в дереве рендеринга — `App` отображает либо `Board`, либо `ResultScreen`, но не оба одновременно.

## User Scenarios
1. **Победа игрока X:** отображается «Победил игрок X!» с кнопками «Играть снова» и «В меню».
2. **Победа игрока O:** отображается «Победил игрок O!».
3. **Ничья:** отображается «Ничья!».
4. **Пользователь нажимает «Играть снова»:** вызывается `onRestart`.
5. **Пользователь нажимает «В меню»:** вызывается `onQuitToMenu`.

## Functional Requirements
- FR-01: Компонент принимает props: `status: GameStatus` (только `won` или `draw`), `onRestart: () => void`, `onQuitToMenu: () => void`. Тип `GameStatus` определён как дискриминированный union:
  ```typescript
  type GameStatus =
    | { kind: 'won'; winner: 'X' | 'O' }
    | { kind: 'draw' }
  ```
- FR-02: При `status.kind === 'won'` отображает «Победил игрок {winner}!», где имя победителя окрашено в соответствующий цвет: `X` — через CSS-переменную `--color-x`, `O` — через CSS-переменную `--color-o`. Обе переменные определены в `src/styles/global.css`.
- FR-03: При `status.kind === 'draw'` отображает «Ничья!».
- FR-04: Отображает кнопку «Играть снова» (`<button>`) — вызывает `onRestart`. Кнопка имеет **primary**-стиль (акцентный, визуально более выраженный).
- FR-05: Отображает кнопку «В меню» (`<button>`) — вызывает `onQuitToMenu`. Кнопка имеет **secondary**-стиль (менее акцентный).
- FR-06: Кнопки используют стандартный `<button>` с понятным текстовым содержимым.
- FR-07: Стили определены в `src/styles/ResultScreen.module.css`.
- FR-08: Создан `src/components/ResultScreen.test.tsx` со smoke-тестами.
- FR-09: Корневой контейнер компонента имеет атрибут `aria-live="assertive"`, чтобы скринридер автоматически объявлял итог партии при появлении компонента.
- FR-10: Кнопки «Играть снова» и «В меню» расположены горизонтально в одну строку; «Играть снова» идёт первой (слева), «В меню» — второй (справа).

## Non-Functional Requirements
- NFR-01: Компонент не хранит локального состояния.
- NFR-02: Компонент корректно отображается при ширине от 320px.

## Boundaries (что НЕ входит)
- Не отображается история ходов партии.
- Не реализуется анимация появления экрана результата.
- Не реализуется счётчик побед между партиями.

## Acceptance Criteria
- [ ] При `status.kind === 'won'` отображается строка «Победил игрок X!» или «Победил игрок O!» в зависимости от значения `winner`.
- [ ] При `status.kind === 'won'` имя победителя (`X` или `O`) окрашено через CSS-переменную `--color-x` или `--color-o` соответственно.
- [ ] При `status.kind === 'draw'` отображается «Ничья!».
- [ ] Кнопка «Играть снова» вызывает `onRestart` при клике.
- [ ] Кнопка «В меню» вызывает `onQuitToMenu` при клике.
- [ ] Обе кнопки — стандартные `<button>` с текстовым содержимым.
- [ ] Кнопка «Играть снова» имеет primary-стиль, кнопка «В меню» — secondary-стиль.
- [ ] Кнопки расположены горизонтально, «Играть снова» — левее «В меню».
- [ ] Корневой контейнер имеет атрибут `aria-live="assertive"`.
- [ ] Smoke-тесты в `ResultScreen.test.tsx` проходят.
- [ ] `npm run lint` не выдаёт ошибок.
