# Specification: status-bar-component

## Context
Компонент `StatusBar` информирует пользователя о текущем состоянии игры: чей ход, кто победил или что наступила ничья. Согласно ADR-010, компонент использует `aria-live="polite"` и `aria-atomic="true"` для объявления изменений скринридерам целиком при каждом изменении. Компонент является чисто презентационным и отображается **только во время активной партии** — до начала игры (на экране настроек) компонент не рендерится вовсе.

## User Scenarios
1. **Игра в процессе:** отображается «Ход игрока X» или «Ход игрока O».
2. **Победа:** отображается «Победил игрок X!» или «Победил игрок O!».
3. **Ничья:** отображается «Ничья!».
4. **Скринридер:** при смене статуса объявляется весь текст нового сообщения целиком через `aria-live="polite"` и `aria-atomic="true"`.
5. **До начала игры:** компонент не рендерится — он отсутствует в DOM на экране настроек.

## Functional Requirements
- FR-01: Компонент принимает props: `status: GameStatus`, где `GameStatus` — дискриминированный union:
  ```typescript
  type GameStatus =
    | { kind: 'playing'; currentPlayer: Player }
    | { kind: 'won'; winner: Player }
    | { kind: 'draw' }
  ```
- FR-02: При `status.kind === 'playing'` отображает текст «Ход игрока {currentPlayer}».
- FR-03: При `status.kind === 'won'` отображает текст «Победил игрок {winner}!».
- FR-04: При `status.kind === 'draw'` отображает текст «Ничья!».
- FR-05: Корневой элемент имеет атрибуты `aria-live="polite"` и `aria-atomic="true"`.
- FR-06: Цвет текста для X — `var(--color-player-x)`, для O — `var(--color-player-o)`, для ничьей — `var(--color-text-secondary)`. Цвет применяется через динамический CSS-класс (`playerX` / `playerO` / `draw`) на корневом элементе компонента.
- FR-07: Стили определены в `src/styles/StatusBar.module.css`.
- FR-08: Создан `src/components/StatusBar.test.tsx` со smoke-тестами для каждого `status.kind` — проверяется, что компонент рендерится без ошибок для каждого из трёх значений `kind`.
- FR-09: Корневым HTML-элементом компонента является `<p>` — семантически подходящий тег для короткого текстового сообщения.
- FR-10: Компонент не рендерится до начала игры; ответственность за его условный рендеринг лежит на родительском компоненте.

## Non-Functional Requirements
- NFR-01: Компонент не хранит локального состояния.
- NFR-02: Текст статуса понятен без визуального контекста (для скринридеров).

## Boundaries (что НЕ входит)
- Не реализуется таймер или счётчик ходов.
- Не реализуется история ходов.
- Не реализуется анимация смены статуса.
- Не реализуется собственная логика скрытия до начала игры — компонент всегда рендерит содержимое при наличии `status` в props; условный рендеринг — забота родителя.

## Acceptance Criteria
- [ ] При `status.kind === 'playing'` отображается корректный текст с именем текущего игрока.
- [ ] При `status.kind === 'won'` отображается имя победителя.
- [ ] При `status.kind === 'draw'` отображается «Ничья!».
- [ ] Присутствует `aria-live="polite"` на корневом элементе.
- [ ] Присутствует `aria-atomic="true"` на корневом элементе.
- [ ] Корневым элементом является тег `<p>`.
- [ ] При `status.kind === 'playing'` и `currentPlayer === 'X'` на корневом элементе применяется CSS-класс `playerX`.
- [ ] При `status.kind === 'playing'` и `currentPlayer === 'O'` на корневом элементе применяется CSS-класс `playerO`.
- [ ] При `status.kind === 'won'` и `winner === 'X'` на корневом элементе применяется CSS-класс `playerX`.
- [ ] При `status.kind === 'won'` и `winner === 'O'` на корневом элементе применяется CSS-класс `playerO`.
- [ ] При `status.kind === 'draw'` на корневом элементе применяется CSS-класс `draw`.
- [ ] Smoke-тесты в `StatusBar.test.tsx` проходят для всех трёх `status.kind` без ошибок рендеринга.
- [ ] `npm run lint` не выдаёт ошибок.
