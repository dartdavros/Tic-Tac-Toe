# Clarifications: status-bar-component

## Question 1
Нужно ли добавить aria-atomic="true" на корневой элемент StatusBar?

*Context:* Без aria-atomic скринридер может объявлять только изменившуюся часть текста, а не всё сообщение целиком — это критично для понятности объявлений при смене хода.

**Options:**
- A) Да, добавить aria-atomic="true" — объявлять весь текст целиком при каждом изменении
- B) Нет, aria-atomic не нужен — достаточно aria-live="polite"
- C) Добавить aria-atomic="true" только для статусов 'won' и 'draw', но не для 'playing'

**Answer:**
Да, добавить aria-atomic="true" — объявлять весь текст целиком при каждом изменении

## Question 2
Какой тип GameStatus используется в props компонента — какова его точная TypeScript-структура?

*Context:* Компонент принимает GameStatus как prop, но структура типа не определена в спецификации — без этого невозможно написать корректную типизацию компонента и тесты.

**Options:**
- A) Дискриминированный union: { kind: 'playing'; currentPlayer: Player } | { kind: 'won'; winner: Player } | { kind: 'draw' }
- B) Объект с опциональными полями: { kind: 'playing' | 'won' | 'draw'; currentPlayer?: Player; winner?: Player }
- C) Тип уже определён в существующем коде проекта — нужно переиспользовать его без изменений

**Answer:**
Дискриминированный union: { kind: 'playing'; currentPlayer: Player } | { kind: 'won'; winner: Player } | { kind: 'draw' }

## Question 3
Как применяется цвет текста согласно FR-06 — к какому именно DOM-элементу и через какой CSS-класс?

*Context:* FR-06 требует разные цвета для X, O и ничьей, но не уточняет: весь текст окрашивается или только имя игрока, и как это реализуется через CSS Modules (динамический класс или inline-стиль).

**Options:**
- A) Весь текст статуса окрашивается через динамический CSS-класс (playerX / playerO / draw) на корневом элементе
- B) Только имя игрока (X или O) оборачивается в <span> с соответствующим классом цвета
- C) Цвет применяется через inline-стиль color на корневом элементе
- D) Весь текст окрашивается через динамический CSS-класс на корневом элементе, имя игрока дополнительно не выделяется

**Answer:**
Весь текст статуса окрашивается через динамический CSS-класс (playerX / playerO / draw) на корневом элементе

## Question 4
Какой HTML-тег использовать как корневой элемент компонента StatusBar?

*Context:* Выбор тега влияет на семантику страницы и доступность — <p>, <div> и <section> имеют разные роли для скринридеров и разное поведение в потоке документа.

**Options:**
- A) <p> — параграф, семантически подходит для короткого текстового сообщения
- B) <div> — нейтральный контейнер без семантики
- C) <section> с aria-label — семантическая секция с явной меткой
- D) <output> — семантически предназначен для результатов вычислений и живых регионов

**Answer:**
<p> — параграф, семантически подходит для короткого текстового сообщения

## Question 5
Что отображать в StatusBar в момент до начала игры, если такое состояние возможно (например, на экране настроек)?

*Context:* Спецификация описывает только три состояния (playing, won, draw), но если GameStatus может быть undefined или иметь kind 'idle', компонент должен обрабатывать этот случай.

**Options:**
- A) StatusBar не рендерится вовсе до начала игры — компонент отображается только во время активной партии
- B) Отображать пустую строку или неразрывный пробел, сохраняя место в layout
- C) Состояние 'idle' не существует — GameStatus всегда имеет один из трёх описанных kind, дополнительная обработка не нужна

**Answer:**
StatusBar не рендерится вовсе до начала игры — компонент отображается только во время активной партии

## Question 6

*Context:* Конституция требует минимум 60% покрытия и обязательные smoke-тесты рендеринга, но не уточняет, достаточно ли проверить только факт рендеринга или нужно также проверять конкретный текст и CSS-классы.

**Options:**
- A) Только smoke: проверить, что компонент рендерится без ошибок для каждого из трёх kind
- B) Smoke + текст: проверить рендеринг и корректность отображаемого текстового содержимого для каждого kind

- D) Полное покрытие: smoke, текст, aria-атрибуты и применение корректных CSS-классов для каждого kind
