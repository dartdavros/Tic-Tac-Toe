# Specification: game-settings-component

## Context
Компонент `GameSettings` является начальным экраном приложения. Пользователь выбирает режим игры и, в режиме PvAI, свою сторону. После подтверждения вызывается `onStartGame` с объектом `GameSettings`. Согласно ADR-010, все элементы формы имеют явные `<label>`. Группы радиокнопок оборачиваются в семантические `<fieldset>` с `<legend>` для соответствия требованиям доступности (a11y).

## User Scenarios
1. **Пользователь выбирает PvP:** нажимает «Начать игру» — вызывается `onStartGame({ mode: 'pvp' })`.
2. **Пользователь выбирает PvAI и сторону X:** нажимает «Начать игру» — вызывается `onStartGame({ mode: 'pvai', humanPlayer: 'X' })`.
3. **Пользователь выбирает PvAI и сторону O:** нажимает «Начать игру» — вызывается `onStartGame({ mode: 'pvai', humanPlayer: 'O' })`.
4. **По умолчанию:** при открытии меню выбран режим PvP.
5. **Пользователь переключается с PvAI обратно на PvP:** выбор стороны сбрасывается до значения по умолчанию (X); блок выбора стороны исчезает из DOM.
6. **Пользователь повторно переключается на PvAI:** блок выбора стороны монтируется заново с выбранной стороной X.

## Functional Requirements
- FR-01: Компонент принимает props: `onStartGame: (settings: GameSettings) => void`.
- FR-02: Отображает заголовок «Крестики-нолики».
- FR-03: Отображает выбор режима: две радиокнопки «Два игрока» (PvP) и «Против компьютера» (PvAI), сгруппированные в `<fieldset>` с `<legend>` и явными `<label>`, связанными через `htmlFor` / `id`.
- FR-04: При выборе режима PvAI отображается дополнительный блок выбора стороны: две радиокнопки «Играть за X» и «Играть за O», сгруппированные в отдельный `<fieldset>` с `<legend>` и явными `<label>`, связанными через `htmlFor` / `id`.
- FR-05: При выборе режима PvP блок выбора стороны **не монтируется в DOM** (условный рендеринг, а не CSS-скрытие).
- FR-06: Кнопка «Начать игру» вызывает `onStartGame` с корректным объектом `GameSettings`.
- FR-07: По умолчанию выбран режим PvP; в режиме PvAI по умолчанию выбрана сторона X.
- FR-08: При каждом переключении режима **на PvAI** выбор стороны сбрасывается до X (независимо от предыдущего значения).
- FR-09: При переключении режима **с PvAI на PvP** текущий выбор стороны сбрасывается до X во внутреннем состоянии компонента, чтобы при следующем переключении на PvAI отображалась сторона X.
- FR-10: Кнопка «Начать игру» всегда активна (`disabled` не используется), так как дефолтные значения формы всегда заданы и все варианты валидны.
- FR-11: Все `<input>` имеют явные `<label>` (связанные через `htmlFor` / `id`).
- FR-12: Стили определены в `src/styles/GameSettings.module.css`.
- FR-13: Создан `src/components/GameSettings.test.tsx` со smoke-тестами.

## Non-Functional Requirements
- NFR-01: Компонент использует локальный `useState` только для управления состоянием формы (выбранный режим и сторона).
- NFR-02: Форма корректно отображается на ширине от 320px.

## Boundaries (что НЕ входит)
- Не реализуется валидация формы (все варианты всегда валидны).
- Не реализуется выбор уровня сложности ИИ.
- Не реализуется история предыдущих игр.

## Acceptance Criteria
- [ ] При выборе PvP и нажатии «Начать игру» вызывается `onStartGame({ mode: 'pvp' })`.
- [ ] При выборе PvAI + X и нажатии «Начать игру» вызывается `onStartGame({ mode: 'pvai', humanPlayer: 'X' })`.
- [ ] При выборе PvAI + O и нажатии «Начать игру» вызывается `onStartGame({ mode: 'pvai', humanPlayer: 'O' })`.
- [ ] Блок выбора стороны отображается только при выборе PvAI (условный рендеринг: при PvP элемент отсутствует в DOM).
- [ ] При переключении с PvAI на PvP и обратно на PvAI блок выбора стороны монтируется с выбранной стороной X.
- [ ] Кнопка «Начать игру» не имеет атрибута `disabled` ни в одном состоянии формы.
- [ ] Группа радиокнопок режима обёрнута в `<fieldset>` с `<legend>`.
- [ ] Группа радиокнопок стороны обёрнута в отдельный `<fieldset>` с `<legend>`.
- [ ] Все `<input>` имеют связанные `<label>` (проверяется через `htmlFor` / `id`).
- [ ] Smoke-тест: компонент монтируется без ошибок.
- [ ] Smoke-тест: вызов `onStartGame` с корректными данными для сценария PvP.
- [ ] Smoke-тест: вызов `onStartGame` с корректными данными для сценария PvAI + X.
- [ ] Smoke-тест: вызов `onStartGame` с корректными данными для сценария PvAI + O.
- [ ] `npm run lint` не выдаёт ошибок.
