# AI_NOTES — Stage 1: Реализация компонента GameSettings и его тестов

## Что было сделано
- Реализован компонент `GameSettings` с полной функциональностью согласно спецификации
- Написаны CSS-стили в `src/components/GameSettings.module.css` с использованием только CSS-переменных из `global.css`
- Написаны smoke-тесты в `src/components/GameSettings.test.tsx`
- Файл `src/styles/GameSettings.module.css` оставлен пустым с комментарием (co-location)

## Почему такой подход

### Управление состоянием формы
Использован `useState` для двух независимых значений: `mode` и `humanPlayer`. Это соответствует NFR-01 — локальный `useState` только для состояния формы. Объединение в один объект было бы избыточным для двух полей.

### Сброс стороны при переключении режима
Функция `handleModeChange` одновременно устанавливает новый режим и сбрасывает `humanPlayer` до `'X'`. Это покрывает оба требования: FR-08 (сброс при переключении на PvAI) и FR-09 (сброс при переключении с PvAI на PvP), так как `handleModeChange` вызывается при любом изменении режима.

### Дискриминированный union GameSettings (ADR-007)
В обработчике `handleSubmit` объект настроек строится через тернарный оператор с явным ветвлением по `mode`. TypeScript корректно сужает тип в каждой ветке, что исключает ошибки компиляции.

### Условный рендеринг блока стороны
Использован `{mode === 'pvai' && (...)}` — блок полностью отсутствует в DOM при PvP (FR-05, ответ на уточнение Q2).

### CSS-стили
Все значения цветов, шрифтов и размеров берутся из CSS-переменных `:root` (инвариант №15). Единственные «хардкодные» значения — отступы в `rem` и `border-radius` в `px`, которые являются структурными, а не дизайн-токенами. Альтернатива: добавить токены `--spacing-*` и `--radius-*` в `global.css`, но это выходит за рамки данной задачи.

### Тесты
Покрыты все четыре обязательных сценария из уточнения Q5:
1. Монтирование без ошибок
2. PvP → `onStartGame({ mode: 'pvp' })`
3. PvAI + X → `onStartGame({ mode: 'pvai', humanPlayer: 'X' })`
4. PvAI + O → `onStartGame({ mode: 'pvai', humanPlayer: 'O' })`

Дополнительно добавлены тесты на: отсутствие `disabled` у кнопки, условный рендеринг блока стороны, сброс стороны при переключении режима, наличие `fieldset`/`legend` (a11y).

### Два файла стилей
Риск из плана реализован: `src/components/GameSettings.module.css` содержит реальные стили, `src/styles/GameSettings.module.css` оставлен пустым с объясняющим комментарием. Компонент импортирует из `./GameSettings.module.css` (co-location, ADR-008).

## Риски и ограничения
- Отступы (`padding`, `gap`, `margin`) заданы в `rem` напрямую, а не через CSS-переменные `--spacing-*`, так как такие токены не определены в `global.css`. При добавлении токенов потребуется рефакторинг.
- `border-radius` задан напрямую (6px, 8px) — аналогично, токены не определены в `global.css`.
- Тест `'кнопка «Начать игру» не имеет атрибута disabled'` проверяет `not.toBeDisabled()` — это корректно для нативного `<button>` без атрибута `disabled`.
