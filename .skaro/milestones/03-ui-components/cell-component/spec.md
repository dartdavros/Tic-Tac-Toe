# Specification: cell-component

## Context
Компонент `Cell` является базовым интерактивным элементом игрового поля. Пользователь кликает на клетку для совершения хода. Согласно ADR-010, клетка должна быть доступна для скринридеров и клавиатурной навигации. Компонент не хранит локального состояния — только отображает переданное значение.

Корневым элементом является нативный `<button>`, что обеспечивает поддержку клавиатурной навигации (Enter, Space) без дополнительных обработчиков `onKeyDown`.

## User Scenarios
1. **Пользователь видит пустую клетку:** отображается пустая область с рамкой, доступная для клика.
2. **Пользователь видит клетку с X:** отображается символ X красного цвета (`--color-player-x`).
3. **Пользователь видит клетку с O:** отображается символ O синего цвета (`--color-player-o`).
4. **Пользователь нажимает Enter или Space на клетке:** вызывается `onCellClick` — нативное поведение `<button>` без дополнительного `onKeyDown`.
5. **Скринридер фокусируется на клетке:** объявляется `aria-label` с позицией и значением в формате `'Клетка {row}×{col}, {пусто | крестик | нолик}'`.

## Functional Requirements
- FR-01: Компонент принимает props: `value: CellValue`, `index: number`, `onCellClick: (index: number) => void`, `disabled?: boolean`. Интерфейс `CellProps` экспортируется из `Cell.tsx`.
- FR-02: Отображает `'X'`, `'O'` или пустую строку в зависимости от `value`.
- FR-03: Корневой элемент — нативный `<button>`. Атрибут `tabIndex` не задаётся вручную — нативная кнопка получает фокус автоматически. При `disabled={true}` передаётся нативный атрибут `disabled` на элемент `<button>`.
- FR-04: Имеет `aria-label` вида `'Клетка {row}×{col}, {значение}'`, где:
  - `row` и `col` вычисляются из `index` (0-based → 1-based): `row = Math.floor(index / 3) + 1`, `col = (index % 3) + 1`;
  - значение: `'пусто'` при `value === null`, `'крестик'` при `value === 'X'`, `'нолик'` при `value === 'O'`.
- FR-05: Обрабатывает `onClick` — вызывает `onCellClick(index)`. Обработчик `onKeyDown` не требуется — нативная кнопка обрабатывает Enter и Space самостоятельно.
- FR-06: При `disabled === true` не вызывает `onCellClick` при клике. Событие клика не останавливается и не перехватывается — просто `onCellClick` не вызывается. Нативный атрибут `disabled` на `<button>` блокирует взаимодействие на уровне браузера.
- FR-07: Стили определены в `src/styles/Cell.module.css`. Цвет X — `var(--color-player-x)`, цвет O — `var(--color-player-o)`.
- FR-08: Размер клетки: `var(--cell-size-desktop)` на десктопе, `var(--cell-size-mobile)` на мобильных (`@media (max-width: 480px)`).
- FR-09: Создан `src/components/Cell.test.tsx` со smoke-тестами.
- FR-10: На корневой элемент `<button>` всегда передаётся `aria-disabled={disabled ?? false}` — явно отражает состояние доступности для вспомогательных технологий независимо от нативного атрибута `disabled`.
- FR-11: CSS-классы применяются по следующей схеме: базовый класс `cell` присутствует всегда; дополнительно применяется один из классов `cellX`, `cellO` или `cellEmpty` в зависимости от значения `value`. Классы берутся из CSS Modules (`Cell.module.css`).
- FR-12: Интерфейс `CellProps` экспортируется именованным экспортом из `Cell.tsx` для использования в родительских компонентах и тестах.

## Non-Functional Requirements
- NFR-01: Компонент не хранит локального состояния (`useState` не используется).
- NFR-02: Компонент корректно отображается при ширине экрана от 320px.

## Boundaries (что НЕ входит)
- Не реализуется анимация появления символа (можно добавить позже через CSS transition).
- Не реализуется подсветка победной линии (это задача компонента `Board`).
- Не реализуется логика определения, можно ли кликнуть на клетку (это задача родительского компонента).
- Не реализуется обработчик `onKeyDown` — нативный `<button>` обеспечивает поддержку клавиатуры без него.

## Acceptance Criteria
- [ ] Компонент рендерится без ошибок для `value = null`, `value = 'X'`, `value = 'O'`.
- [ ] Корневым элементом является нативный `<button>`, а не `<div>`.
- [ ] Присутствует `role="button"` не требуется явно — нативная кнопка имеет его по умолчанию; тест проверяет наличие элемента через `getByRole('button')`.
- [ ] `aria-label` для `value = null` содержит строку `'пусто'` (например, `'Клетка 1×1, пусто'`).
- [ ] `aria-label` для `value = 'X'` содержит строку `'крестик'` (например, `'Клетка 1×1, крестик'`).
- [ ] `aria-label` для `value = 'O'` содержит строку `'нолик'` (например, `'Клетка 1×1, нолик'`).
- [ ] `aria-disabled={false}` присутствует при `disabled` не передан или `disabled={false}`.
- [ ] `aria-disabled={true}` присутствует при `disabled={true}`.
- [ ] Клик вызывает `onCellClick` с корректным `index`.
- [ ] Нажатие Enter вызывает `onCellClick` (нативное поведение `<button>`).
- [ ] Нажатие Space вызывает `onCellClick` (нативное поведение `<button>`).
- [ ] При `disabled={true}` клик не вызывает `onCellClick`.
- [ ] При `disabled={true}` событие клика не останавливается (не вызывается `stopPropagation`).
- [ ] Базовый класс `cell` присутствует на элементе всегда.
- [ ] При `value = 'X'` присутствует класс `cellX`, отсутствуют `cellO` и `cellEmpty`.
- [ ] При `value = 'O'` присутствует класс `cellO`, отсутствуют `cellX` и `cellEmpty`.
- [ ] При `value = null` присутствует класс `cellEmpty`, отсутствуют `cellX` и `cellO`.
- [ ] Интерфейс `CellProps` экспортируется именованным экспортом из `Cell.tsx`.
- [ ] Smoke-тесты в `Cell.test.tsx` проходят.
- [ ] `npm run lint` не выдаёт ошибок.
