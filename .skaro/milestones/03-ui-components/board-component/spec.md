# Specification: board-component

## Context
Компонент `Board` отображает игровое поле и делегирует клики наверх через `onCellClick`. Он является контейнером для 9 компонентов `Cell` и управляет их состоянием `disabled` на основе текущего статуса игры. Согласно ADR-010, поле имеет `role="grid"` и `aria-label`. Клетки структурированы в три строки с полным ARIA-деревом: `role="grid"` → `role="row"` → `role="gridcell"`.

## User Scenarios
1. **Начало игры:** отображается пустое поле 3×3, все клетки доступны для клика.
2. **В процессе игры:** занятые клетки отображают символы, свободные — доступны для клика.
3. **Победа:** победные клетки визуально выделены изменением фона через CSS-переменную `--color-win-highlight`.
4. **Игра завершена:** все клетки недоступны для клика (`disabled`).

## Functional Requirements
- FR-01: Компонент принимает props: `board: BoardState`, `onCellClick: (index: number) => void`, `winLine?: number[]`, `disabled?: boolean`.
- FR-02: Рендерит ровно 9 компонентов `Cell` с корректными `value` и `index`.
- FR-03: Использует CSS Grid (`grid-template-columns: repeat(3, 1fr)`) для расположения клеток. Визуальное разделение клеток реализуется через свойство `gap` на контейнере `Board` — без использования `border` на самих клетках `Cell`.
- FR-04: Передаёт `disabled={true}` в `Cell`, если `disabled === true` или клетка занята (`board[index] !== null`). Занятая клетка всегда заблокирована независимо от значения prop `disabled` — `onCellClick` для неё не вызывается никогда.
- FR-05: Если `winLine` передан, клетки с индексами из `winLine` получают дополнительный CSS-класс для визуального выделения. Выделение реализуется изменением фона клетки через CSS-переменную `--color-win-highlight` (значение по умолчанию: `#f1c40f`).
- FR-06: Корневой элемент компонента имеет `role="grid"` и `aria-label="Игровое поле"`. Внутри рендерятся три `div` с `role="row"`, каждый из которых содержит три компонента `Cell` с `role="gridcell"`, формируя полное ARIA-дерево grid.
- FR-07: Стили определены в `src/styles/Board.module.css`. CSS-переменная `--color-win-highlight` объявлена в глобальных стилях (`src/styles/global.css`) и используется в классе выделения победной клетки.
- FR-08: Создан `src/components/Board.test.tsx` ровно с двумя smoke-тестами: рендер 9 клеток и вызов `onCellClick` с корректным индексом при клике на свободную клетку.

## Non-Functional Requirements
- NFR-01: Компонент не хранит локального состояния.
- NFR-02: Корректно отображается при ширине экрана от 320px.

## Boundaries (что НЕ входит)
- Не реализуется логика определения победителя (это задача `gameLogic`).
- Не реализуется анимация ходов.
- Не реализуется нумерация строк/столбцов поля.
- Не используется SVG-линия поверх поля для выделения победной комбинации.
- Не используется `border` на компоненте `Cell` для визуального разделения клеток.

## Acceptance Criteria
- [ ] Рендерится ровно 9 компонентов `Cell`.
- [ ] Клик на свободную клетку вызывает `onCellClick` с корректным индексом.
- [ ] При `disabled={true}` ни одна клетка не вызывает `onCellClick`.
- [ ] Клик на занятую клетку не вызывает `onCellClick` независимо от значения prop `disabled`.
- [ ] Клетки из `winLine` имеют CSS-класс визуального выделения; фон таких клеток определяется переменной `--color-win-highlight` (`#f1c40f`).
- [ ] Присутствует `role="grid"` и `aria-label="Игровое поле"` на корневом элементе.
- [ ] Присутствуют три `div` с `role="row"`, каждый содержит три элемента с `role="gridcell"`.
- [ ] Визуальное разделение клеток реализовано через `gap` на контейнере, а не через `border` на `Cell`.
- [ ] Smoke-тесты в `Board.test.tsx` проходят (ровно два теста: рендер 9 клеток и вызов `onCellClick`).
- [ ] `npm run lint` не выдаёт ошибок.
