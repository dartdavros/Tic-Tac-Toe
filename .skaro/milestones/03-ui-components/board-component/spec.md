# Specification: board-component

## Context
Компонент `Board` отображает игровое поле и делегирует клики наверх через `onCellClick`. Он является контейнером для 9 компонентов `Cell` и управляет их состоянием `disabled` на основе текущего статуса игры. Согласно ADR-010, поле имеет `role="grid"` и `aria-label`.

## User Scenarios
1. **Начало игры:** отображается пустое поле 3×3, все клетки доступны для клика.
2. **В процессе игры:** занятые клетки отображают символы, свободные — доступны для клика.
3. **Победа:** победные клетки визуально выделены (другой фон или рамка).
4. **Игра завершена:** все клетки недоступны для клика (`disabled`).

## Functional Requirements
- FR-01: Компонент принимает props: `board: BoardState`, `onCellClick: (index: number) => void`, `winLine?: number[]`, `disabled?: boolean`.
- FR-02: Рендерит ровно 9 компонентов `Cell` с корректными `value` и `index`.
- FR-03: Использует CSS Grid (`grid-template-columns: repeat(3, 1fr)`) для расположения клеток.
- FR-04: Передаёт `disabled={true}` в `Cell`, если `disabled === true` или клетка занята (`board[index] !== null`).
- FR-05: Если `winLine` передан, клетки с индексами из `winLine` получают дополнительный CSS-класс для визуального выделения.
- FR-06: Имеет `role="grid"` и `aria-label="Игровое поле"`.
- FR-07: Стили определены в `src/styles/Board.module.css`.
- FR-08: Создан `src/components/Board.test.tsx` со smoke-тестами: рендер 9 клеток, вызов `onCellClick`.

## Non-Functional Requirements
- NFR-01: Компонент не хранит локального состояния.
- NFR-02: Корректно отображается при ширине экрана от 320px.

## Boundaries (что НЕ входит)
- Не реализуется логика определения победителя (это задача `gameLogic`).
- Не реализуется анимация ходов.
- Не реализуется нумерация строк/столбцов поля.

## Acceptance Criteria
- [ ] Рендерится ровно 9 компонентов `Cell`.
- [ ] Клик на свободную клетку вызывает `onCellClick` с корректным индексом.
- [ ] При `disabled={true}` ни одна клетка не вызывает `onCellClick`.
- [ ] Клетки из `winLine` имеют визуальное выделение (CSS-класс).
- [ ] Присутствует `role="grid"` и `aria-label="Игровое поле"`.
- [ ] Smoke-тесты в `Board.test.tsx` проходят.
- [ ] `npm run lint` не выдаёт ошибок.

## Open Questions
- Нужно ли добавить разделительные линии между клетками через CSS или использовать `border` на самих клетках?
- Как визуально выделять победную линию: изменение фона клетки, цвета символа или отдельная SVG-линия поверх поля?
