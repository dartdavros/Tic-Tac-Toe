# Specification: app-component

## Context
Компонент `App` является корневым компонентом приложения. Он владеет единственным `useReducer` с `gameReducer` и `INITIAL_STATE`, управляет условным рендерингом трёх экранов и передаёт `dispatch` в дочерние компоненты через callback-функции. Согласно архитектурным инвариантам, все мутации состояния происходят только через `dispatch`.

## User Scenarios
1. **Первый запуск:** пользователь видит экран меню (`GameSettings`).
2. **Начало игры:** пользователь настраивает игру и нажимает «Начать» — отображается игровое поле (`Board` + `StatusBar`).
3. **Завершение партии:** после победы или ничьей автоматически отображается `ResultScreen`.
4. **«Играть снова»:** пользователь нажимает кнопку — возврат в меню.
5. **«В меню»:** пользователь нажимает кнопку — возврат в меню.
6. **Ошибка рендеринга:** `ErrorBoundary` перехватывает ошибку и показывает `ErrorFallback`.

## Functional Requirements
- FR-01: `App` использует `useReducer(gameReducer, INITIAL_STATE)` как единственный источник состояния.
- FR-02: При `state.screen === 'menu'` рендерится `<GameSettings onStartGame={...} />`.
- FR-03: При `state.screen === 'game'` рендерится `<Board ... />` и `<StatusBar ... />`.
- FR-04: При `state.screen === 'result'` рендерится `<ResultScreen ... />`.
- FR-05: `onStartGame` диспатчит `{ type: 'START_GAME', payload: settings }`.
- FR-06: `onCellClick` диспатчит `{ type: 'MAKE_MOVE', payload: { index } }`.
- FR-07: `onRestart` диспатчит `{ type: 'RESTART' }`.
- FR-08: `onQuitToMenu` диспатчит `{ type: 'QUIT_TO_MENU' }`.
- FR-09: Всё дерево компонентов ниже `App` обёрнуто в `<ErrorBoundary fallback={<ErrorFallback onReset={handleReset} />}>`.
- FR-10: `handleReset` диспатчит `{ type: 'RESTART' }` и сбрасывает `ErrorBoundary`.
- FR-11: `Board` получает `disabled={state.status.kind !== 'playing'}` для блокировки ходов при завершённой партии.
- FR-12: `Board` получает `winLine` из `state.status` при `status.kind === 'won'`.
- FR-13: Создан `src/components/App.test.tsx` со smoke-тестом начального рендера (экран меню отображается без ошибок).
- FR-14: Стили `App` определены в `src/styles/App.module.css` (центрирование, максимальная ширина контейнера).

## Non-Functional Requirements
- NFR-01: Первый рендер приложения происходит без ошибок в консоли.
- NFR-02: Переход между экранами не вызывает мигания или промежуточных состояний.

## Boundaries (что НЕ входит)
- Не реализуется роутинг (URL не меняется при смене экрана).
- Не реализуется анимация переходов между экранами.
- Не реализуется персистентность состояния между сессиями.

## Acceptance Criteria
- [ ] При запуске приложения отображается экран меню.
- [ ] После нажатия «Начать игру» отображается игровое поле.
- [ ] После завершения партии отображается экран результата.
- [ ] «Играть снова» и «В меню» возвращают в меню.
- [ ] `ErrorBoundary` оборачивает всё дерево компонентов.
- [ ] Smoke-тест в `App.test.tsx` проходит (начальный рендер, экран меню).
- [ ] `npm run lint` не выдаёт ошибок.
- [ ] `npm run build` завершается без ошибок.
